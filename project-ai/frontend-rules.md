# FRONTEND RULES

## STACK

- React
- Next.js
- TypeScript
- TailwindCSS

## COMPONENTES

- reutilizáveis
- desacoplados
- pequenos
- responsivos

## RESPONSIVIDADE

Garantir:

- mobile first
- tablet
- desktop
- telas ultrawide

## PERFORMANCE

Evitar:

- re-renderizações desnecessárias
- carregamentos pesados
- imagens sem otimização

## UX

Sempre utilizar:

- loading states
- empty states
- skeleton loading
- feedback visual
- animações suaves

## ESTADO

Utilizar:

- hooks customizados
- context apenas quando necessário
- evitar prop drilling

## FORMULÁRIOS

Sempre validar:

- frontend
- backend
- mensagens claras

## ACESSIBILIDADE

Garantir:

- contraste adequado
- navegação por teclado
- aria labels
- semântica HTML

## SEO

Garantir:

- metadata
- Open Graph
- estrutura semântica
- performance Lighthouse
